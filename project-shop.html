Verdant Plant Shop — a catalog-style storefront that loads fast, even image-heavy.

Clientlocal plant & garden shop Rolefrontend developer Timeline2.5 weeks

A product catalog lives and dies by its images — but image-heavy pages are usually the slowest. The challenge was making a browsable, photo-rich shop that still loads quickly on an average connection.

the problem

A catalog that took 8 seconds to show a single leaf.

Verdant's previous site loaded every product photo at full resolution, all at once. On mobile data, the homepage alone could take the better part of ten seconds — long enough for most visitors to give up before seeing a single plant.

the approach

Treat performance as a feature, not an afterthought.

Every product image is compressed and served at the size it's actually displayed, with lazy-loading so only visible images load first. Filters (light needs, pet-safe, price) let visitors narrow the catalog instead of scrolling through everything.

  • Image pipeline: compressed, modern formats, explicit width/height to prevent layout shift
  • Client-side filtering with plain JavaScript — no page reloads
  • Grid that reflows cleanly from 4 columns down to 1 on mobile
the build

HTML & CSS doing the heavy lifting.

The catalog grid is pure CSS Grid with no layout library needed. JavaScript is reserved for filtering logic and the product quick-view modal, keeping the page light and fast to parse.
