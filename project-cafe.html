Brew & Co. Café — a menu-first site built to turn scrollers into walk-ins.

Clientindependent café, single location Rolefrontend developer Timeline2 weeks

The brief was simple: most visitors check the menu and hours on their phone while standing outside, so the site needed to answer those two questions in under three seconds.

the problem

A beautiful Instagram, an outdated website.

Brew & Co. had loyal regulars and a strong social following, but their old site was a slow, unstyled template with an out-of-date menu PDF. New visitors couldn't tell if they were open, what they served, or where to park — so they left.

the approach

Design for the phone in someone's hand.

I started from the mobile layout, not the desktop one. The hero leads with hours and a map pin. The menu is rendered as real HTML, not a PDF, so it's searchable, fast, and easy for the café to update themselves.

  • Sticky "Order & hours" bar that stays visible while scrolling
  • Menu organized by daypart (breakfast / lunch / pastries) instead of one long list
  • Compressed, lazy-loaded photography so the page stays fast on mobile data
the build

Clean markup, no bloat.

Built with semantic HTML and hand-written CSS — no heavy frameworks for a page this size. A small JavaScript layer handles the menu filters and the mobile nav. Every image ships in a modern format with explicit sizing to avoid layout shift.
